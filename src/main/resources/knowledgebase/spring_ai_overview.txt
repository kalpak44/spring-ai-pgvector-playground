Spring AI Overview

Spring AI is a framework within the Spring ecosystem that simplifies building AI-powered applications in Java. It provides a unified abstraction layer over multiple AI providers, allowing developers to swap models without changing application code.

Core Concepts

ChatClient
The central entry point for interacting with large language models (LLMs). ChatClient supports a fluent builder API for constructing requests, attaching advisors, and streaming responses. It wraps provider-specific clients (Ollama, OpenAI, Anthropic, etc.) behind a consistent interface.

ChatMemory
Spring AI provides built-in conversation memory through the ChatMemoryAdvisor. Memory can be stored in-memory, in a JDBC-backed repository, or in a vector store. When using JDBC memory, Spring AI automatically creates the required schema and persists message history per conversation ID.

Advisors
Advisors are middleware components that intercept and enrich ChatClient calls. They follow the chain-of-responsibility pattern. Common advisors include:
- MessageChatMemoryAdvisor: injects conversation history into each request
- QuestionAnswerAdvisor: performs RAG retrieval and injects context documents
- SimpleLoggerAdvisor: logs requests and responses for debugging

Supported Models
Spring AI 1.0 supports the following providers out of the box:
- OpenAI (GPT-4o, GPT-4 Turbo)
- Anthropic (Claude 3.5 Sonnet, Claude 3 Opus)
- Ollama (local models: Llama 3, Gemma 3, Mistral, Phi-3)
- Azure OpenAI
- Google Vertex AI (Gemini)
- Amazon Bedrock
- Mistral AI

Embedding Models
Spring AI abstracts embedding generation through the EmbeddingModel interface. Embeddings are numerical vector representations of text used for semantic similarity search. Ollama supports local embedding models such as mxbai-embed-large (1024 dimensions) and nomic-embed-text (768 dimensions).

Configuration
Spring AI uses Spring Boot auto-configuration. Each provider has a dedicated starter dependency and properties prefix:
- spring.ai.ollama.base-url
- spring.ai.ollama.chat.model
- spring.ai.ollama.embedding.model
- spring.ai.openai.api-key
- spring.ai.anthropic.api-key

Vector Store Integration
Spring AI integrates with multiple vector databases through the VectorStore interface:
- PgVector (PostgreSQL extension)
- Redis
- Pinecone
- Weaviate
- Milvus
- Chroma

The VectorStore interface exposes add(), delete(), and similaritySearch() methods. The QuestionAnswerAdvisor uses similaritySearch() to retrieve relevant documents and inject them into the prompt as context.
